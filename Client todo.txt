1. Получить базу данных, с которой можно работать.
	+ Получить данные с API.
	+ Получить ссылки на карты.
	+ Парс.
	+ Закинуть в базу данных.

2. Создать интерфейс главного меню.
	- Вход.
		- Логин пароль.
		- Регистрация.
		- Редирект на меню.
	- Кнопки.
		- Играть.
		+ Редактор колод.
		- Настройки.
		- Язык.
		- Выход.
		+ Парс.

3. Создать редактор колод, который получает эту базу данных в виде массива объектов и может свободно работать с ним на клиенте.
	+ Интерфейс.
		+ Область превью.
		+ Область кнопок.
		+ Область колоды.
		 + Основная колода.
		 + ЛРИГ колода.
		+ Область поиска.
	- Красивый интерфейс.
		- Позже.
	- Код.
		+ Получение базы данных.
		+ Вывод данных из базы данных.
			+ Настройка фильтров.
				+ по имени.
				+ по цвету.
				+ по типу.
				+ по тексту.
				+ по ограничению.
				+ по таймингу (арты).
				+ по уровню (лриги, сигни).
				+ по лимиту (лриги, сигни).
				+ по классу или лриг-типу (сигни ).
				+ по формату игры (кс, ас).
			+ Максимальное число выводимых карт (увеличивается по клику на кнопку).
		+ Добавление карт в колоду.
		+ Удаление карт из колоды.
		+ Наведение на карту курсором.
		+ Разделение добавляемых карт на основную колоду и ЛРИГ-колоду.
		- Сохранение колод.
			+ Создать на основе объекта deck, объект сохранения, в котором записаны номера карт.
			+ Сохранить его на сервер.
			- Сказать об удачном сохранении.
		+ Загрузка колод.
			+ Найти файлы на сервере, отправить на клиент список колод.
			+ После выбора пройтись по файлу и закинуть в колоду все карты на основе переданных номеров карт.
			+ Отобразить это.
		+ Создание новой колоды.
			+ Очистить текущую колоду и отобразить это.
		- Удаление колоды.
			+ Показать список колод.
			- По клику по одной из колод, спросить, хочет ли юзер таки удалить ее.
			+ Если да - удалить колоду.
			- Если нет - закрыть окно.
			+ Обновить список.
			- Сказать об удачном удалении.
		- Переписать систему сохранения колод и настроек пользователей.
4. Создать систему авторизации и аутентикации.
	+ Создать объект юзера и схему для базы данных.
	+ Научиться пользовать и ввести в систему jwt.
	+ Логин и пароль.
	+ Регистрация.
	+ Проверка авторизации.
	+ Логаут.
5. Лобби.
	- Основной интерфейс лобби (доступный при первом входе на страницу):
		- Блок со списком игр.
			- Список игр хранится в стэке (массиве) на сервере и выводится на страницу на ее основе.
			- Чтобы создать комнату, пользовать испускает событие, обработчик которого на сервере добавит комнату в стэк.
			- Чтобы удалить комнату, необходимо, чтобы созданную комнату покинули все. Это инициирует событие на сервере, убирающее комнату из стэка.
			- Каждое изменение стэка сразу же инициирует событие на клиенте, которое получает новый список игр, которым нужно заменить существующий.
		- Фильтр по списку игр.
			- По имени.
			- По заполненности.
			- По наблюдателям.
			- По юзеру.
			- По наличию пароля.
			- По состоянию комнаты*. (игра началась, ожидание начала)
		- Блок с чатом.
			- Список пользователей в лобби.
			- Отображаются имена пользователей из базы данных (его можно менять в профиле).
			- Аватарки**
			- Вкладка событий сервера, которые необходимо сообщать пользователю*
		- Блок с кнопками.
			- Создать игру. Ведет на модаль с формой создания.
			- Присоединиться к игре. Активна только если выбрана игра из списка. Опционально выводит модаль, требующую пароль.
			- Наблюдать за игрой. Активна только если выбрана игра из списка, за которой можно наблюдать.
			- Список друзей с их статусом***. Ведет на модаль со списком.
			- Редактор колод.
			- Вернуться в меню.




- Баги.
	- Спецсимволы в именах отображаются неверно.
	- Поиск по имени не ищет имена со спецсимволами.

- Предполагаемые баги.
	- поиск по тексту не будет искать большие предложения, потому что внутри объекта карты они поделены тегами и другим лишним текстом.
	- поиск по цвету плохо ищет карты, если запросить более одного цвета.
	- поиск по сету может плохо работать.
